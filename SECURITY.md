# 安全指南

## 敏感信息管理

### 1. 环境变量
所有敏感信息（密码、Token、密钥）必须通过 `.env` 文件管理，不要硬编码到代码中。

### 2. 配置文件
以下文件包含敏感信息，已添加到 `.gitignore`：
- `ecosystem.config.cjs` - PM2 配置（使用 `.example` 模板）
- `central-hub/config/hub.json` - Hub 配置
- `config/hub.json` - 全局配置

### 3. API 安全

#### 当前状态
Central Hub API (端口 51000) 目前**无认证保护**，仅适用于内网环境。

#### 生产环境建议
1. **网络隔离**：确保 51000 端口仅在内网可访问
2. **反向代理**：通过 Nginx/Caddy 添加 Basic Auth
3. **防火墙规则**：限制访问 IP 段

#### 未来改进
- [ ] 添加 API Token 认证
- [ ] 添加 JWT 认证
- [ ] 添加请求频率限制

### 4. CORS 配置

当前配置允许所有来源 (`origin: "*"`)，生产环境应限制为：
```json
{
  "cors": {
    "enabled": true,
    "origin": ["http://192.168.9.200:51000", "http://localhost:51000"]
  }
}
```

### 5. 日志安全
- 日志中不应包含完整的密码、Token
- 当前代码已对敏感信息做脱敏处理（如 `sun-token...`）

## 报告安全问题

如发现安全漏洞，请通过以下方式报告：
- 创建 GitHub Issue（标记为 Security）
- 或直接联系项目维护者

## 安全检查清单

部署前请确认：
- [ ] `.env` 文件已创建且包含所有必需变量
- [ ] `ecosystem.config.cjs` 不包含硬编码凭据
- [ ] 51000 端口仅在内网可访问
- [ ] CORS 配置已收紧
- [ ] 日志文件权限正确（600 或 640）
- [ ] 数据目录权限正确（700 或 750）
